R&D Execution Monitoring Dashboard for the Radio-Electronics Industry of the Russian Federation

Disclaimer

To respect confidentiality, this demo uses fully synthetic data that mirrors the structure and edge cases of the real project. The original dataset was much larger and significantly messier: inconsistent data types, no unified schema, mixed date/number formats, missing values, duplicates, and frequent operator typos.

R&D Progress Dashboard

What I used on the real project

  • Excel 2016 + Power Query for data ingestion and cleaning
  • A small Python script (pandas) to automatically build the list of organizations
  • Dashboard delivered in Power BI

Original dashboard scope

In the production version I built a ~5-page dashboard to track work at a granular level: overdue items, on-time performance, and risk of schedule slippage. It also included operational quality views for staff—surfacing duplicates, data errors, and operator typos for quick correction and cross-checks. Budgeting was covered in depth (by region, by organization, and by activity type), alongside rollups and drill-downs for management. The public Tableau demo here shows the core logic on synthetic data; the full internal version was broader and more detailed.

Why Tableau here

For public sharing I rebuilt the dashboard in Tableau with synthetic data. This lets me keep company information private while showing the same analytical logic — and it highlights my ability to switch tools and deliver quickly.
